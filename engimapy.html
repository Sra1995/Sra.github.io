EnigmaPy

Description

EnigmaPy brings the historical Enigma machine to life in a modern Python script. The Enigma machine, famously known for its use by the German military during World War II, employed a complex system of rotors, reflectors, and plugboards to encode and decode messages. In this project, users have the opportunity to interact with a simplified Enigma machine through a terminal interface.

The main components of the Enigma machine, such as rotors, reflectors, and plugboards, are represented as classes in the Python script. Users can load rotor configurations from files, set up initial rotor positions, add plugboard pairs, and perform encryption and decryption on messages.

Algorithm

  
1. Rotor Class:
    - Initialize rotor with a given wiring configuration and starting position.
    - Implement forward and backward methods for character encoding and decoding.

2. Reflector Class:
    - Initialize reflector with a given wiring configuration.
    - Implement a reflect method for character reflection.

3. Plugboard Class:
    - Initialize plugboard with an initial mapping of characters to themselves.
    - Implement methods to add, view, and remove plugboard pairs.
    - Process method to apply plugboard mapping to characters.

4. EnigmaMachine Class:
    - Initialize Enigma machine with rotor filenames and reflector wiring.
    - Implement methods to load rotors, set rotor positions, add plugboard 
    - pairs, view plugboard, and perform encryption/decryption.

5. User Interface (Main Function):
    - Display a cool ASCII art header for the Enigma machine.
    - Present a menu for users to:
        * Set up rotors and initial positions.
        * Add, view, and remove plugboard pairs.
        * Encrypt and decrypt messages.
        * Exit the Enigma machine.

6.Cool Animation Function:
    -Implement a function for printing text letter by letter with a specified 
     delay for a cool animation effect.

 Libraries:
    - Ensure that the following Python packages are installed:
      - `os`            # to allow interaction with the operating system
      - `time`          # to allow time-related operations
      - `csv`           # to allow interaction with .csv files
      - `string`        # ascii_uppercase

Algorithm Analysis

The Enigma machine simulation operates with a time complexity that is predominantly linear. Loading rotor configurations from files and setting initial rotor positions have linear time complexity proportional to the number of rotors. The encryption and decryption processes iterate through the characters of the input message, resulting in linear time complexity relative to the length of the message.

The plugboard operations, including adding, viewing, and removing pairs, operate in constant time, as they involve simple dictionary manipulations. Overall, the simulation is designed for simplicity and ease of understanding, making it suitable for educational purposes and cryptographic exploration in a Python environment.

The plugboard, also known as the Steckerbrett, allowed users to swap pairs of letters before and after the main encryption process in the Enigma machine. The full list of options for the plugboard is essentially all possible pairs of letters. Since each plugboard pair involves two letters, there are a total of 26 × 25 / 2= 325 . 26×25/2=325 possible pairings.
