EnigmaPy

Description

EnigmaPy brings the historical Enigma machine to life in a modern Python script. The Enigma machine, famously known for its use by the German military during World War II, employed a complex system of rotors, reflectors, and plugboards to encode and decode messages. In this project, users have the opportunity to interact with a simplified Enigma machine through a terminal interface.

The main components of the Enigma machine, such as rotors, reflectors, and plugboards, are represented as classes in the Python script. Users can load rotor configurations from files, set up initial rotor positions, add plugboard pairs, and perform encryption and decryption on messages.

Algorithm

  
1. Rotor Class:
    - Initialize rotor with a given wiring configuration and starting position.
    - Implement forward and backward methods for character encoding and decoding.

2. Reflector Class:
    - Initialize reflector with a given wiring configuration.
    - Implement a reflect method for character reflection.

3. Plugboard Class:
    - Initialize plugboard with an initial mapping of characters to themselves.
    - Implement methods to add, view, and remove plugboard pairs.
    - Process method to apply plugboard mapping to characters.

4. EnigmaMachine Class:
    - Initialize Enigma machine with rotor filenames and reflector wiring.
    - Implement methods to load rotors, set rotor positions, add plugboard 
    - pairs, view plugboard, and perform encryption/decryption.

5. User Interface (Main Function):
    - Display a cool ASCII art header for the Enigma machine.
    - Present a menu for users to:
        * Set up rotors and initial positions.
        * Add, view, and remove plugboard pairs.
        * Encrypt and decrypt messages.
        * Exit the Enigma machine.

6.Cool Animation Function:
    -Implement a function for printing text letter by letter with a specified 
     delay for a cool animation effect.

 Libraries:
    - Ensure that the following Python packages are installed:
      - `os`            # to allow interaction with the operating system
      - `time`          # to allow time-related operations
      - `csv`           # to allow interaction with .csv files
      - `string`        # ascii_uppercase

Algorithm Analysis

The Enigma machine simulation operates with a time complexity that is predominantly linear. Loading rotor configurations from files and setting initial rotor positions have linear time complexity proportional to the number of rotors. The encryption and decryption processes iterate through the characters of the input message, resulting in linear time complexity relative to the length of the message.

The plugboard operations, including adding, viewing, and removing pairs, operate in constant time, as they involve simple dictionary manipulations(self.mapping). Overall, the simulation is designed for simplicity and ease of understanding, making it suitable for educational purposes and cryptographic exploration in a Python environment.




Counting the Possible Plugboard Settings

The formula used for calculating the number of possible plugboard pairings is based on the concept of combinations and permutations. Here's a breakdown of the formula:

n!
---------
((n-2m)! * m! * 2^m)

- n!: Represents the factorial of n, which is the product of all positive integers up to n.
- (n-2m)!: Represents the factorial of (n-2m), which is used to account for the permutations of unpaired letters after 2m letters have been paired.
- m!: Represents the factorial of m, accounting for the permutations of the paired letters.
- 2^m: Represents the number of ways the paired letters can be connected (since each pair can be connected in two ways).

The formula is derived from the idea that for each step in choosing pairs, the number of available choices decreases. Here's how it is applied:

  1. Choose 2 letters for the first pair (C(n,2)).
  2. For each of these, choose 2 letters for the second pair (C(n-2,2)).
  3. Repeat this process m times.
  4. Multiply by 2^m for the two possible connections for each pair.

When n represents the number of distinct objects (letters), and m represents the number of pairs, the formula calculates the total number of ways to arrange the letters when 2m are paired.

For example, in the case of m = 10 pairs and n = 26 letters, the formula calculates the number of ways to arrange the letters after 10 pairs have been connected, considering the permutations and combinations involved.

Possible Combinations of Plugboard Pairs:

For 1 pair (m = 1, n = 26): (26! / ((26-2*1)! * 1! * 2^1)) = (26! / (24! * 1! * 2)) = 325
For 5 pairs (m = 5, n = 26): (26! / ((26-2*5)! * 5! * 2^5)) = (26! / (16! * 5! * 32)) = 5,019,589,575
For 10 pairs (m = 10, n = 26): (26! / ((26-2*10)! * 10! * 2^10)) = (26! / (6! * 10! * 2^10)) = 150,738,274,937,250
For 11 pairs (m = 11, n = 26): (26! / ((26-2*11)! * 11! * 2^11)) = (26! / (4! * 11! * 2^11)) = 205,552,193,096,250
For 12 pairs (m = 12, n = 26): (26! / ((26-2*12)! * 12! * 2^12)) = (26! / (2! * 12! * 2^12)) = 102,776,096,548,125



So why did they use 10 and not 11?

During World War II, the German Enigma machines typically used a plugboard with 10 pairs of letters connected. This means that 10 pairs of letters were swapped before and after the signal passed through the rotors and reflector. The reason for using 10 pairs was likely a balance between increasing the complexity of the encryption and practical considerations such as ease of use for the machine operators.

Some might wonder why the number of possible combinations is so large, but start to decreases when hitting 12 pairs and above. The number of possible plugboard settings decreases when using 12 pairs compared to 11 pairs because the formula for calculating the number of combinations involves dividing by a factor that increases more rapidly.
